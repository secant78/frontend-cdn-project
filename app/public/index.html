Architecture Overview

🌐

CloudFront CDN

Global edge network with 400+ PoPs. Caches static assets at the edge, routes dynamic requests to Elastic Beanstalk origin.

  • Cache behavior: /assets/* → S3
  • Cache behavior: /* → EB origin
  • HTTPS enforced (redirect HTTP)
⚙

Elastic Beanstalk

Managed Node.js platform with auto-scaling EC2 instances behind an Application Load Balancer.

  • Node.js 18 runtime
  • ALB with health checks on /health
  • Auto-scaling: 1–4 instances
🔒

WAF (Web ACL)

AWS-managed rule groups attached to CloudFront. Blocks OWASP Top 10 attacks before traffic ever reaches the origin.

  • AWSManagedRulesCommonRuleSet
  • AWSManagedRulesKnownBadInputsRuleSet
  • Rate limiting: 2,000 req / 5 min
💾

S3 Static Assets

Private S3 bucket serving CSS, JS, images. Only accessible through CloudFront via Origin Access Control (OAC).

  • OAC — no public bucket access
  • Versioned bucket for rollbacks
  • Cache-Control: 1-year TTL

Security Layers

WAF SQL Injection Protection AWSManagedRulesSQLiRuleSet blocks malicious queries at the edge
WAF XSS Protection CommonRuleSet filters cross-site scripting payloads
WAF Rate Limiting IP-based rate limiting prevents brute-force and DDoS
WAF IP Reputation AmazonIpReputationList blocks known bad actors automatically
CF HTTPS Enforced All HTTP traffic redirected to HTTPS at the CloudFront edge
S3 Origin Access Control S3 bucket is private; only CloudFront service principal has read access
EB Security Groups EC2 instances only accept traffic from the CloudFront managed prefix list
EB Helmet.js Headers CSP, HSTS, X-Frame-Options, and other security headers set at the app layer
